# CPU 高负载场景

## 场景描述

CPU 使用率持续过高是数据库性能问题中最常见的异常类型之一。本场景模拟多种导致 CPU 资源消耗过高的典型情况，用于验证诊断引擎对 CPU 相关问题的识别和根因分析能力。

## 包含用例

| 用例ID | 用例名称 | 难度 | 描述 |
|--------|----------|------|------|
| case_01_sensors | 98个传感器并发插入 | Easy | 大量并发插入操作导致CPU飙升 |
| case_02_compute | 复杂计算导致CPU飙升 | Medium | 复杂SQL计算任务占用大量CPU |

## 典型特征

- CPU 使用率持续 > 80%
- 系统负载 (Load Average) 异常升高
- 进程状态显示大量 CPU 密集型操作
- 可能伴随 I/O wait 增高

## 常见根因

1. **全表扫描**：缺少索引导致大量数据扫描
2. **复杂计算**：SQL 中包含大量聚合、排序、函数计算
3. **并发写入**：大量并发 INSERT/UPDATE 操作
4. **锁等待**：锁竞争导致 CPU 空转
5. **连接风暴**：短时间内大量连接请求

## 诊断要点

1. 检查 `pg_stat_activity` 中的活跃查询
2. 分析慢查询日志找出 CPU 消耗大的 SQL
3. 检查执行计划是否存在全表扫描
4. 检查索引使用情况
5. 检查并发连接数和锁等待情况
