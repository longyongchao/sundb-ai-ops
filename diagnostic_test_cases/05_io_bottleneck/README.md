# IO 瓶颈场景

## 场景描述

磁盘 I/O 瓶颈是数据库性能问题的常见原因，特别是在数据量大的场景下。本场景模拟 I/O 相关的性能问题，验证诊断引擎对 I/O 问题的识别能力。

## 包含用例

| 用例ID | 用例名称 | 难度 | 描述 |
|--------|----------|------|------|
| case_07_checkpoint | 检查点风暴 | Hard | 频繁检查点导致IO飙升 |

## 典型特征

- 磁盘 I/O 等待时间高
- iowait 占比高
- 查询响应时间波动大
- 磁盘吞吐量达到上限

## 常见根因

1. **检查点配置不当**：checkpoint_timeout 或 checkpoint_completion_target 设置不合理
2. **WAL 写入过快**：大量写入导致 WAL 积压
3. **共享缓冲区命中率低**：频繁磁盘读取
4. **磁盘性能不足**：IOPS 或吞吐量达到上限

## 诊断要点

1. 检查 `pg_stat_bgwriter` 视图
2. 分析检查点频率和持续时间
3. 检查 WAL 写入量
4. 分析缓冲区命中率
5. 检查磁盘 I/O 指标
