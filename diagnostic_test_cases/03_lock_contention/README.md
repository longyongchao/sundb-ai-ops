# 锁竞争场景

## 场景描述

锁竞争是数据库并发控制中的常见问题，严重时会导致死锁，造成事务回滚和业务中断。本场景模拟锁竞争和死锁问题，验证诊断引擎对锁相关问题的识别和处理能力。

## 包含用例

| 用例ID | 用例名称 | 难度 | 描述 |
|--------|----------|------|------|
| case_05_deadlock | 死锁问题 | Hard | 多事务交叉锁定导致死锁 |

## 典型特征

- 事务长时间等待
- 锁等待队列增长
- 死锁检测告警
- 应用端超时错误

## 常见根因

1. **交叉更新**：多个事务以不同顺序更新相同资源
2. **长事务**：事务持有锁时间过长
3. **热点数据**：大量事务竞争同一行或表
4. **锁升级**：行锁升级为表锁
5. **外键约束**：外键检查导致的隐式锁

## 诊断要点

1. 检查 `pg_locks` 视图中的锁等待情况
2. 分析死锁日志找出涉及的事务和SQL
3. 检查长事务和空闲事务
4. 分析热点数据访问模式
5. 检查外键约束和索引情况
