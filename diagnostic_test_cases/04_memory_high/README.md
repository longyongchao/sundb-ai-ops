# 内存高使用场景

## 场景描述

内存使用过高会导致数据库性能下降，严重时触发OOM Killer。本场景模拟内存消耗过大的典型情况，验证诊断引擎对内存问题的识别能力。

## 包含用例

| 用例ID | 用例名称 | 难度 | 描述 |
|--------|----------|------|------|
| case_06_sort_mem | 排序内存溢出 | Medium | 大数据量排序导致内存不足 |

## 典型特征

- 内存使用率持续 > 85%
- 大量临时文件产生
- Swap 使用增加
- 查询响应时间变长

## 常见根因

1. **work_mem 设置过大**：每个排序操作分配过多内存
2. **连接数过多**：每个连接消耗内存累加
3. **大结果集返回**：查询返回大量数据
4. **内存泄漏**：某些操作未正确释放内存

## 诊断要点

1. 检查 PostgreSQL 进程内存使用
2. 分析 work_mem 和连接数配置
3. 检查临时文件使用情况
4. 分析慢查询中的排序操作
