# 慢查询场景

## 场景描述

慢查询是数据库性能问题中最常见且影响最大的异常类型。本场景模拟多种导致查询执行缓慢的典型情况，用于验证诊断引擎对慢查询问题的识别、根因分析和优化建议生成能力。

## 包含用例

| 用例ID | 用例名称 | 难度 | 描述 |
|--------|----------|------|------|
| case_03_full_scan | 全表扫描慢查询 | Medium | 缺少索引导致全表扫描 |
| case_04_missing_index | 缺失索引导致查询缓慢 | Hard | 复杂查询缺少关键索引 |

## 典型特征

- 查询执行时间超过阈值（默认10秒）
- 大量顺序扫描（Seq Scan）
- 高磁盘读取量
- 可能伴随 CPU 或 I/O 资源消耗

## 常见根因

1. **缺少索引**：查询字段没有合适的索引
2. **索引失效**：索引未被正确使用
3. **全表扫描**：WHERE 条件无法利用索引
4. **统计信息过期**：优化器选择错误的执行计划
5. **锁等待**：查询被其他事务阻塞

## 诊断要点

1. 分析 `pg_stat_statements` 找出慢查询
2. 使用 `EXPLAIN ANALYZE` 分析执行计划
3. 检查索引使用情况和覆盖率
4. 检查统计信息更新时间
5. 检查是否存在锁等待
